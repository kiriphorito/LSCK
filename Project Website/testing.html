Testing


In the page we will be discussing about possible testing methods that we can use during our software development.

There are two main types of testing that we hope to do in this project. Unit testing and integration & functional testing with the possible help of automated testing.

Unit Testing

Unit testing is where you test the components of the code base to ensure that they are function correctly and to the way you expect. Doing this early on means that you can weed out bugs and potentials errors earlier and know that it isn’t an issue in high level testing such as integration and functional testing.

In terms of the extension, we have decided that it would be best to code it in C# rather than Visual Basics. As a result, the NUnit framework we feel is the most appropriate for unit testing in C# and it is actually a port from JUnit, the unit testing framework for Java. With the help of automated testing, this should be run every time changes are made to the code base.

Integration and Functional Testing/User Acceptance Testing

This type of testing is where you test the entire system as a whole to see if the product meets the requirements set by the client and ensure the product works as it should. Unfortunately, we haven’t yet been able to look into a way of doing this automatically. But manually, we can ask other members of the COMP204P and COMP205P who use Visual Studio in their development to load the plug-in and see if the extension works in the way that they expect.

This time could have been spent further developing the project and an extension could have generated the website. All you had to do was link the files and add tags to the code that you wanted to include as a snippet. This is what we want to investigate and achieve.

Cross Browser Testing (HTML Generation)

For the HTML generation section of the extension, we need to test and see if the generated website works on a variety of different browser and check for unexpected behaviours and fix them.

We hope to test the generated websites on the following browsers:

  • Chrome (PC & Android)
  • Firefox
  • Safari (Mac & iOS)
  • Opera
  • Microsoft Edge

There are a number of tools available on the internet that has some form of automation when it comes to cross browser testing but they come at a price. We will look further into this to find a suitable framework for this.

Speed and UX Testing (HTML Generation)

For speed an user interface testing, we can us Google’s Pagespeed Insights and as well as telling us speed and UX issues, it can also tell us how to improve it. We can also use Pingdom to tell us the exact loading times of the HTML pages. Of course, this is after hosting it on an Azure server.
