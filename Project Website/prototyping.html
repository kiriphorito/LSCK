Prototyping


In this page we will talk about our prototyping and our methods

Platform

The sole potential platform for the extension is Windows, as the extension is intended to work on Visual Studio only, and at the moment macOS is only compatible with VS Code. Initially, the application will work with as many languages possible (C#, Java, C…), as we understand all businesses utilize different language combinations for their projects.

Process

Initially, we explored different extensions that would serve as an inspiration or provide us with ideas in the process of developing the extension. It was also important to understand how extensions are made, and we tried creating some basic ones from scratch first, to make sure we truly understood what we were doing. On top of this, we researched into how extensions are packaged into VSIX format and uploaded to the visual studio marketplace. The team found code-search extensions and even open source examples. The latter helped us understand how to create files. So far, we have developed an extension which can create a file, for either C# or Java, and save selected code by the user under different sections (also created by the user). This is an important milestone in our progress, as it is one of the 3 key functionalities we are willing to have in our final product. The design of the extension still looks very trivial, and we hope to work on that too in the next stages of the project:



To develop this basic prototype the team also had to research how to read and write files properly, as the extension needs to read every file saved (for every language) to list the different sections created, and also has to write the code selected at each specific section, accompanied by a comment. We have also studied open source extensions which are helpful to scan the entire code in the project, and will be very useful for the next steps, as we wish to search an entire visual studio open project for specific sequences of characters.
