Development Plan

Iteration 1 – Creating the Back End

After researching and discussing, each team member came up with two different approaches. Both only doing simple tasks so that we don’t spend too much time on an approach that we were no longer going to use. After some discussion we decided to use the FJController, JSON and FileHandler approach. We continued down this path using unit testing to see if the system works the way we thought it should. The other approach as a text file for each language with certain delimiters representing a certain detail.


Iteration 2 – Development of WebsiteGenerator Class

With the back-end now completed we can now move onto the WebsiteGenerator class which would use the back-end to get the data needed to create the portfolio. First we made the website example what of what we hope it would look like and asked colleagues for their opinions and feedback.

Iteration 3 – Creating the Front End

With the back-end and WebsiteGenerator class now finished we started to develop the front-end of the extension. Using the designs and initial prototype from the first term we started to develop the user interface. It went through several iterations after asking colleagues for their opinions on the extension.

Iteration 4 – Development of Extractor and SSH Classes

We started to develop the Extractor and SSH classes in parallel since they are relatively small classes. These would be the last set of classes for the feature set.

Iteration 5 – Optimisations for Visual Studio

During this phase, we mainly did optimisations on the extension such as using threading to enhance the user experience so that Visual Studio wouldn’t hang or freeze when doing long tasks such as uploading a website or generating a preview. Also bug fixing is certain situations when the extension is being used such as opening LSCK when no solution is open. Or opening up a new solution doesn’t reset the JSON data.

Iteration 6 – User Interface Enhancements

This is the last phase for programming and it was just to enhance the user interface to be more colour co-ordinated with themes in Visual Studio as well as layouts.
