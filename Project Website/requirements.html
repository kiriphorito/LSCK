Requirements and Use Cases


In this page we will discuss about the brief that was set, the requirments that were negotiated and the use cases that the team has come up with.

Initial Requirements

With the first team meeting, we were giving a brief by Lee Stott which is as follows:

This is intended to simplify business processes for new startups / small team practitioners. A discussion with startups/tech entrepreneurs should elicit what other tools would make their life easier. By end of a project, a complete news publication site should be generated to help promote a project.

Baseline outcomes: A series of tools for cloud account migration , a way to import project contract documents and link to sections of VSTS repositories as a notation, a news notification for subscribers to a project in the form of a website

Gathering Requirements

After receiving the initial requirements from Lee Stott, we proceeded get into contact with the client as well as preparing questions to ask the client to clarify the brief and come up with a set of requirements that we find is reasonable.

For the requirements, we used the FURPS framework to break up the requirements into groups which are Functional Requirements, Usability Requirments, Reliabilty Requirments, Performance Requirements and Supportability requirments.

Then for each of the requirements, we used the MoSCoW method to prioritise the different requirements.

Negotiated Requirements

After coming up with the requirements we gave this to the client and negotiated the requirements that suited both the client and the team.

Use Cases

Use Case AddCodeAndComment
ID UC1
Relevant Requirement F1 & F2
Description The user adds code to the portfolio and writing a description with it
Primary Actors Developer/User
Secondary Actors
Preconditions The user has written code on Visual Studio IDE
Main Steps
  1. The user finds the code that they wish to add to the document
  2. Using the extension in Visual Studio, they add a comment in the textbox that they wish to associate with the snippet
  3. The user then highlights the code that they wish to save as a snippet
  4. The user then clicks on the save button
Post Conditions The extension saves the snippet of code and the associated text in a separate file.
Alternative Steps
  1. The user finds the code that they wish to add to the document
  2. Then using the associated tags with the language, they add the tags around the part of the desired code
  3. Including in the tags is the comment that the developer wishes to associate with the snippet

Use Case GenerateHTMLPages
ID UC2
Relevant Requirement F3
Description The user wishes to generate the HTML portfolio
Primary Actors Developer/User
Secondary Actors
Preconditions The user has added snippets of code via the extension, uploading other documents or by adding tags directly to the code
Main Steps
  1. The user has looked through all the snippets of code and other documents that they wish to publish
  2. The user then clicks on the generate button
Post Conditions The extension generates a set of HTML files with its dependencies, ready to upload
Alternative Steps

Use Case UploadOtherDocuments
ID UC3
Relevant Requirement F5
Description The user wishes add other documents to the portfolio such as videos and presentations
Primary Actors Developer/User
Secondary Actors
Preconditions The user has documents ready to be part of the portfolio stored locally
Main Steps
  1. The user clicks on upload on the extension
  2. They then browse through the file system of the computer to find the files they wish to include in the portfolio
  3. They then click upload
  4. The user then places the document in a certain position in the portfolio
  5. They then add a comment
  6. Finally, they click on save
Post Conditions The extension will save the directory to the file and the comment in a separate file ready for retrieval
Alternative Steps

Use Case UploadWebsite
ID UC4
Relevant Requirement F4
Description The user wishes upload the generated website to Azure
Primary Actors Developer/User
Secondary Actors
Preconditions The user has already generated a HTML portfolio from a selection of snippets of code and documents. Related to GenerateHTMLPages use case
Main Steps
  1. The user enter in their credentials for the Azure web service
  2. The user then clicks on upload to push the generated HTML onto an instance
Post Conditions The extensions returns a link to which the user can give to anyone.
Alternative Steps

Use Case DeletingCodeOrDocument
ID UC5
Relevant Requirement F7
Description The user no longer wishes to include a document or snippet of code in the portfolio
Primary Actors Developer/User
Secondary Actors
Preconditions The user has already tagged or added the document that they wish to remove from the portfolio
Main Steps
  1. The user looks at the extension to see if the code of document has included in the portfolio. May want to use the SearchAdded use case (UC7)
  2. The user then clicks on the item that they wish to remove
Post Conditions If the snippet was added via, tags the tags will be removed. If the snippet of document was added via the extension, then its entry in a separate document will be removed
Alternative Steps
  1. The user looks at what the extension has included in the portfolio
  2. The user manually removes the tags themselves

Use Case AddPages
ID UC6
Relevant Requirement F8
Description The user wishes to segregate is information into different pages in the portfolio
Primary Actors Developer/User
Secondary Actors
Preconditions None required
Main Steps
  1. The user clicks on add page
  2. The user then adds a name to the page
Post Conditions A new separate file is created for the new page
Alternative Steps
