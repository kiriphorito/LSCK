SDK & Future Developments


In this page we will talk what potential the extension has for the future.

SDK and Collabration

Many parts of our project could be applied for many other different purposes external to the actual LSCK extension. For the purpose of the latter, we developed a singleton called FJController, which can be used to write and read different parts of a JSON files. With some slight modifications, FJController could be used to store and retrieve data from JSON files so that you can have a small database structure with each entry possible having a file associated with it, or with even less modifications, it could be used to generate even more complex websites. You could even redesign the entire front-end, the UI and just use FJController for the back end. The modularty of the MVC design pattern allows these developments to happen.

We also developed an extractor class which goes through an entire solution looking for specific character sequences. With some changes, the extractor class could be used for other purposes like a search tool, or to retrieve specific data from files based on what was found within them.

Potential Future Developments

Even though we are very please with the extension. We could fulfill all the requirements set by the client in time. And with any extension, there are always more features you can add to allow the user to have a more pleasing and enjoyable experience.

Future development could include:

  • Allowing the user to use other operating systems for hosting such as Windows
  • Deploying a virtual machine from within the extension using Subscription ID with Azure
  • Making generated websites with PHP so that they can be modified easily by experienced web developers
  • Maybe add some backend control for things like content databases or authentication
  • Maybe linking the extension to other microsoft services like VSTS
  • Allows more customisations to the user such as having a side-bar rather than just a menu bar
